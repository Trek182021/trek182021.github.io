Introduction

Gnome sort is a sorting algorithm originally proposed by Iranian computer scientist Hamid Sarbazi-Azad (professor of Computer Science and Engineering at Sharif University of Technology) in 2000. The sort was first called stupid sort (not to be confused with bogosort), and then later described by Dick Grune and named gnome sort.

The gnome sort is similar to insertion sort in a way that it works with one item at a time but gets the item to the proper place by a series of swaps, similar to a bubble sort. It is conceptually simple, requiring no nested loops.

Step Process

Gnome Sort also called Stupid sort is based on the concept of a Garden Gnome sorting his flower pots. A garden gnome sorts the flower pots by the following method-

  • He looks at the flower pot next to him and the previous one; if they are in the right order he steps one pot forward, otherwise he swaps them and steps one pot backwards.
  • If there is no previous pot (he is at the starting of the pot line), he steps forwards; if there is no pot next to him (he is at the end of the pot line), he is done.

Algorithm

Here is pseudocode for the gnome sort using a zero-based array:

                    --> Function/Method accepts an array
                    procedure gnomeSort(a[]):

                    --> Start in position one
                    pos := 0

                    --> While the position is not equal to the size of array. 
                    while pos < length(a):

                        -->  If the position is in the first spot of the array,
                        -->  If number in spot we're in is greater than the number behind,
                        -->  Move one space
                        if (pos == 0 or a[pos] >= a[pos-1]):
                            pos := pos + 1

                        -->  Otherwise, Swap value in present position and the previous value.
                        else:
                            swap a[pos] and a[pos-1]
                            pos := pos - 1

Facts about Gnome Sort

Limitations of Gnome Sort:

  • Since Gnome Sort is not quite the fastest sorting algorithm, its not used very much, but it could a good starting point in understanding how other sorting algorithm works.
  • An example would be, the insertion sort which is one of the basic sorting algorithms which works the same way as gnome sort without the going back step whenever a swap takes place.
  • The average running time of this algorithm is O(n^2) but tends towards O(n) if the list is nearly sorted.
